Javascript Basics

How does Javascript compare to HTML and CSS?

HTML and CSS for the most part are very static, they decide the layout and appearance of a site but very little about what the site does. What happens when you push buttons? Where does the information go when you fill out a form? What if you need the site to do something for you when you ask it.

Javascript is a language that controls almost all of these active aspects of the site and is very different code to look at. It's like the nervous system of your body. All the thinking happens there and it's the source of all action on your website.

Control Flow

Everytime a piece of javascript code get's run (executed) in the browser, the browser is running one instruction at a time. Most of the time this just means the lines of code are executed in order.

With loops and functions it is possible to repeat some code until a certain condition is achieved, or jump to another section of code entirely, do what it says there, then jump back. The flow of control is the order or pattern in which the lines of code are run. Let's use waking up and eating breakfast as an example, maybe a morning looks like this:

  1. Wake up.
  2. Get up and stretch.
  3. Repeat this until not hungry:
    1. Put piece of bread in toaster.
    2. Spread butter and marmite on toast.
    3. Eat toast
  4. Do the brushing teeth routine.
  5. Is it a weekday? If yes, get ready for work. Otherwise read a book.
  6. Brushing teeth routine:
    1. Put toothpaste on toothbrush
    2. Brush teeth
    3. Rinse mouth with water
    4. Go back to what you were doing before

If we start at the top and follow the instructions, clearly they aren't all going to happen in the order they're written. At 3. eating breakfast the flow of control goes into a loop until a condition (hunger) is satisfied. Later the flow of control jumps from breakfast to the teeth brushing routine and back to where it left off, in real application a section of code like this is called a function, when a function is called all the instructions inside it get run. Later on we get a point in the code that looks like it's going to branch into two options, weekday or weekend. So these are some ways that real flow of control in code might work.

Arrays and Objects

These are both ways to store a collection of data. An array is a set of items in a specific order, you refer to an item (element) in the array by its index. The index is the position of the item in the array starting at 0 so if you want the 20th item in an array you ask for array[19]. If you want the first item - array[0].

Data inside objects are stored with labels, so when you access an item you ask for it by it's label. You might want to know the number of seats stored in the object 'car', you'd refer to it with: car.numberOfSeats

Functions

A function is a collection of code with it's own name that can be called - requested from somewhere else in the script. Functions can take input values or objects and can also return values. For example you might be writing some code in which you frequently need to find the greatest number in a list of numbers. You could write a function findGreatestNumber(list) {code that does the job}. Now that it's written you can ask for this to happen from anywhere in the script.
